# Загальний огляд проекту: Розподіл завдань між розробниками

## Аналіз поточного проекту

**Проект:** Django REST API для моніторингу цін (Silpo Monitor)

**Поточна структура:**
- Django 4.2+ з Django REST Framework
- Моделі: `ProductType`, `Store`, `Product`, `BeerProduct`
- REST API з базовими CRUD операціями
- Web інтерфейс для перегляду продуктів

---

## Завдання лабораторної роботи

### Основна частина (75%)
1. Створити 6 агрегованих запитів (ORM Django)
2. Створити REST API ендпоїнти з pandas DataFrame
3. Виконати статистичний аналіз даних
4. Створити 12 графіків (6 × Plotly + 6 × Bokeh)
5. Інтегрувати інтерактивні дашборди в Django

### Додаткова частина (25%)
1. Реалізувати паралельні обчислення (multithreading/multiprocessing)
2. Знайти оптимальні параметри доступу до БД
3. Запустити 100-200 запитів паралельно
4. Створити графіки продуктивності на дашборді

## РОЗРОБНИК 1: Аналітичні запити та REST API

### Основні обов'язки
**Напрямок:** Backend розробка, Django ORM, REST API, pandas аналіз

### Завдання
1. **Створити 6 агрегованих запитів** в `monitoring/repositories/analytics.py`
2. **Створити 6 REST API ендпоїнтів** з pandas DataFrame
3. **Статистичний аналіз:** створити `monitoring/analytics/statistical_analysis.py`
4. **Документація:** Порівняння Django ORM vs pandas, API документація
5. **Тестування:** Unit тести для репозиторіїв (12 тестів), API тести (6 тестів)

**Час:** ~30 годин  
**Складність:** ⭐⭐⭐⭐

## РОЗРОБНИК 2: Візуалізація даних Plotly та Дашборд v1

### Основні обов'язки
**Напрямок:** Backend/Frontend розробка, Plotly візуалізація, Django templates, JavaScript

### Завдання

5. **Тестування:** Unit тести для графіків (6 тестів), інтеграційні тести (5 тестів)

## РОЗРОБНИК 3: Візуалізація Bokeh + Паралельні обчислення

### Основні обов'язки

5. **Візуалізація продуктивності** (3 графіки: час виконання, throughput, heatmap)
